## 1. 개요

### 목적
적도의(Equatorial Mount)를 사용하는 천문 관측자가 AstroHopper를 더 정확하고
편리하게 사용할 수 있도록 두 가지 기능을 추가한다.

### 대상 사용자
- 독일식·포크식 적도의 사용자
- 극축 정렬이 완료된 수동/자동 추적 적도의 사용자

### 해결하는 문제
| 문제 | 기존 상태 | 개선 후 |
|------|-----------|---------|
| 정렬 정확도 | 1성 정렬만 가능 — 정렬 별에서 멀어질수록 오차 누적 | 2성 정렬로 전천 정확도 향상 |
| 좌표 표시 | 방향 화살표만 표시 | ΔRA / ΔDec 수치(도/분) 추가 표시 |

## 3. 사용 방법

### 3-1. 적도의 모드 활성화
1. 설정창(⚙) 열기
2. **EQ Mount Mode** 체크박스 켜기 → 저장됨

### 3-2. 1성 정렬 (기존과 동일)
1. 망원경을 밝은 별에 조준
2. **Align** 버튼 클릭
3. 지도에서 해당 별 탭 → 3초 카운트다운 후 정렬 완료
4. 상태 표시: `Aligned ✓` + **2nd ✦** 버튼 등장

### 3-3. 2성 정렬 (EQ 모드 전용)
1. 망원경을 첫 번째 별과 멀리 떨어진 두 번째 밝은 별에 조준
2. **2nd ✦** 버튼 클릭
3. 지도에서 두 번째 별 탭 → 잔차 보정 적용

> **팁**: 두 별이 RA·Dec 방향으로 각각 20° 이상 떨어져 있을수록 보정 효과가 큼

### 3-4. ΔRA/ΔDec 오프셋 확인
- 목표 천체를 탭하면 화면 하단에 표시:
  ```
  ΔRA: +2.3°    ΔDec: -1.1°
  ```
- 1도 미만일 때는 분(') 단위로 표시: `ΔRA: +45'`
- EQ 마운트 RA·Dec 미동 손잡이를 해당 방향으로 조작

## 4. 기술 상세

### 4-1. 2성 정렬 수학

기존 `selectAlign()`은 매 정렬 시 `global_align_matrix`를 덮어씀:
```
M = matMul(daz_mat, dalt_mat)   // 매번 새 행렬
global_align_matrix = M
```

2성 정렬에서는 1성 보정 행렬(M1)에 잔차 보정(M_residual)을 합산:
```
// 1성 정렬 후 getCameraRays()가 M1-보정된 벡터를 반환
// 2성 정렬에서 계산된 correction은 M1-좌표계의 잔차

global_align_matrix = matMul(M_residual, M1)
//  전체: raw_sensor → M1 보정 → M_residual 보정 → 하늘 좌표
```

### 4-2. ΔRA/ΔDec 계산 (역변환 공식)

ENU 단위 벡터 `fwd = [e, n, u]` → 고도/방위각 → H(시각) → RA:

```javascript
// 고도·방위각 (표준: 북=0°, 동=90°)
alt  = asin(u)
az_n = atan2(e, n)

// 적위
sinDec = sin(lat)*sin(alt) + cos(lat)*cos(alt)*cos(az_n)
Dec = asin(sinDec)

// 시각 (Meeus Chapter 13 역변환)
H = atan2(
    -sin(az_n)*cos(alt),
    -cos(az_n)*sin(lat)*cos(alt) + sin(alt)*cos(lat)
)

// 적경
LST = 2π*(0.7790572732640 + 1.00273781191135448*tu) + lon_rad
RA = (LST - H) * 180/π  (mod 360)
```

## 6. 향후 과제 (Phase 3)

### 6-1. 극축 정렬 보조 (미착수)
극축 정렬이 얼마나 틀렸는지 계산해 보여주는 기능.
- 2성 정렬 결과 행렬에서 극축 오차를 역산
- "극축: 북쪽으로 2.3° 어긋남" 등으로 표시

### 6-2. 적도의 좌표 기반 탐색
현재 방향 화살표 대신 "RA 손잡이 +30° / Dec 손잡이 -5°" 형식 표시.
→ ΔRA/ΔDec 표시가 이미 구현됨. 화살표 방향 보완 정도만 필요.

### 6-3. 적도의 추적 속도 보상
추적 중 생기는 alpha_diff 드리프트를 항성시 기반으로 자동 보상.
(현재는 재정렬로 대응 가능하므로 우선순위 낮음)
